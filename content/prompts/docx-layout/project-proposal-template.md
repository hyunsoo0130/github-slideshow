promptText: |
  Create a Word document layout for a project proposal titled "[PROJECT TITLE]"
  for client [CLIENT NAME], submitted by [YOUR NAME/COMPANY].

  STRUCTURE AND FORMAT:

  COVER PAGE
  - Proposal title: "[PROJECT TITLE]" — large, centered, bold
  - "Prepared for: [CLIENT NAME]" and "Prepared by: [YOUR NAME]"
  - Submission date, version number
  - Status badge: "CONFIDENTIAL PROPOSAL" in a colored box

  1. EXECUTIVE SUMMARY (half page)
  - Problem we're solving for [CLIENT NAME]
  - Our proposed approach in 3 sentences
  - Expected outcome/benefit
  - Investment summary (one line)

  2. PROBLEM STATEMENT
  - Clear definition of [CORE PROBLEM]
  - Current pain points (3 bullet points)
  - Business impact if problem remains unsolved
  - Supporting data/statistics placeholder: [INDUSTRY STAT]

  3. PROPOSED SOLUTION
  - Solution overview paragraph
  - Key features/deliverables as a numbered list (5 items)
  - Differentiation from alternatives: 2-column comparison table
    Column A: [YOUR APPROACH] | Column B: Standard Approach

  4. PROJECT TIMELINE
  - Gantt-style text table with phases:
    Phase 1: Discovery — [DURATION]
    Phase 2: Development — [DURATION]
    Phase 3: Testing — [DURATION]
    Phase 4: Launch — [DURATION]
    Phase 5: Post-launch support — [DURATION]
  - Key milestones highlighted in bold

  5. INVESTMENT & PRICING
  - Itemized table:
    | Service Item | Hours/Units | Rate | Total |
  - Subtotal, Tax ([TAX RATE]%), Grand Total row in bold
  - Payment terms: [PAYMENT TERMS]
  - Validity: This proposal valid for 30 days

  6. TEAM
  - For each team member: Name, Role, 2-sentence bio, key relevant experience
  - Use a 2-column layout: photo placeholder left, bio right

  7. NEXT STEPS
  - 3 clear action items with owner and deadline
  - Call to action: "To proceed, please sign and return by [DATE]"

  8. SIGNATURE PAGE
  - Signature blocks for both parties
  - Date field, printed name field, title field

  Typography: Professional yet approachable. Use [ACCENT COLOR] for headers and accents.
  Generate realistic placeholder content throughout so I can see the full layout.
usage: |
  1. Paste into Claude with all variables filled in
  2. Claude returns structured text — copy sections into Word
  3. Apply your company's brand colors to headers
  4. Replace placeholder content with real project details
  5. Add actual team photos and real statistics
  6. Export as PDF before sending to clients
variables: |
  [PROJECT TITLE] → Name of the proposed project
  [CLIENT NAME] → Client/company receiving the proposal
  [YOUR NAME/COMPANY] → Your name or agency name
  [CORE PROBLEM] → The problem you're solving
  [DURATION] → Time for each phase (e.g., "2 weeks", "1 month")
  [TAX RATE] → Applicable tax percentage
  [PAYMENT TERMS] → e.g., "50% upfront, 50% on delivery"
  [ACCENT COLOR] → Your brand color
tips: |
  • Ask Claude to fill in realistic example pricing for your industry
  • Add "include case study placeholder from similar project" in Section 3
  • Request "risk assessment table with mitigation strategies" for enterprise clients
  • For creative projects: add "mood board description and visual direction section"
  • Add "warranty and support terms section after pricing" for software projects
